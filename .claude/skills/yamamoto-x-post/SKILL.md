# 山本さん X投稿生成スキル

テーマを受け取り、Xリサーチ（任意）→ 記事構成 → RAG検索 → X投稿文6パターン生成 を行う。
各ステップでユーザーの承認を得てから次に進む。
承認は必ず **AskUserQuestionツール** を使い、選択肢付きで確認すること。

## 設定

- **insightsフォルダ**: `/Users/teach/environment/app/2507/260406_yamamoto-rag-x/insights/`
- **Xリサーチ**: デフォルトON（ユーザーが「Xリサーチなし」「リサーチスキップ」等と言った場合はOFF）

## X API セットアップ（初回のみ）

X APIを利用するには、Pay-Per-Use（従量課金）プランの登録が必要。
2026年2月にFreeプランが廃止され、Pay-Per-Useプランが新設された。

### 前提条件
- 有効なX(Twitter)アカウント
- クレジットカード（最低$5のチャージ）

### 登録手順

**STEP 1: 旧Developer Portalでアカウント作成 & Bearer Token取得**
1. https://developer.x.com にアクセスしXアカウントでサインイン
2. アカウント名・利用目的（250文字以上）を入力して送信
3. 「Projects & Apps」から自分のAppの「Keys and tokens」タブを開く
4. 「Bearer Token」セクションの「Generate」をクリックしてトークンをコピー

**STEP 2: 新Developer ConsoleでPay-Per-Useプランに切り替え**
5. https://console.x.com にアクセス（旧Portalとは別の画面）
6. 左メニュー「アプリ」を開く
7. STEP 1で作ったAppが「Free」になっている場合、**Pay-Per-Useに変更**する
   ※ Freeのままだと検索APIで「403 Client Forbidden」エラーになる

**STEP 3: クレジット購入 & 支出上限設定**
8. 左メニュー「請求書作成」→「クレジット」を開く
9. 「クレジットを購入する」から最低$5（約750円）をチャージ
10. **「支出上限を管理」ボタンから支出上限を$5に設定**（重要！）
    ※ デフォルトは「無制限」。設定しないとクレジット超過分が追加請求される

**STEP 4: 環境変数の設定**
11. ターミナルで以下を実行:
```bash
echo 'export X_API_BEARER_TOKEN="ここにSTEP 1でコピーしたトークンを貼り付け"' >> ~/.bashrc
source ~/.bashrc
```

**STEP 5: 動作確認**
12. ターミナルで以下を実行し、トークンが表示されればOK:
```bash
echo $X_API_BEARER_TOKEN
```

### 参考リンク
- 旧Developer Portal: https://developer.x.com
- 新Developer Console: https://console.x.com
- 詳しい手順ガイド: https://www.and-and.co.jp/gas-lab/x-twitter-api-start/
- 503エラー解決（Free→Pay-Per-Use移行）: https://www.and-and.co.jp/gas-lab/x-twitter-api-503-error/

> **料金目安（Pay-Per-Use）:** 検索・データ取得は投稿より安価。支出上限を必ず設定すること。
> 設定しないとクレジット超過分が追加請求される可能性あり。

## ステップ② Xで最新情報をリサーチ

> Xリサーチ=OFFの場合はこのステップをスキップし、ステップ③も省略してステップ④へ進む。

**やること:**

3段階のフォールバック方式でXの最新情報を収集する。
各Phaseで十分な結果（具体的な投稿内容を含む3件以上）が得られたら、次のPhaseはスキップする。

### Phase 1: WebSearchで検索（無料）
1. WebSearchツールを使い、以下の検索クエリでXの最新投稿・トレンドを調査する:
   - `{テーマ} site:x.com`
   - `{テーマ} Twitter トレンド`
   - テーマに関連するキーワードの組み合わせ（2〜3パターン）
2. **判定:** 具体的な投稿内容を含む結果が3件以上 → Phase 2, 3をスキップ

### Phase 2: Nitterでフォールバック補完（無料）
3. Phase 1の結果が不十分な場合、Nitter経由で追加リサーチする:
   - WebFetchツールで `https://xcancel.com/search?q={テーマ}&f=tweets` を取得
   - 取得できない場合は `https://twiiit.com/search?q={テーマ}&f=tweets` を試す
   - それでも取得できない場合は `https://nitter.privacydev.net/search?q={テーマ}&f=tweets` を試す
4. **判定:** Phase 1 + 2 の合計で3件以上 → Phase 3をスキップ

> **Note:** Nitterインスタンスは不安定なため、全インスタンスがダウンしている場合がある。

### Phase 3: X API で最終補完（従量課金）
5. Phase 1 + 2 でも情報が不十分な場合、X APIで検索する:
   - 環境変数 `X_API_BEARER_TOKEN` が設定されているか確認
   - 未設定の場合: ユーザーに「X APIが未設定のため、現在の情報で進めます。APIを使いたい場合はセットアップ手順を参照してください」と通知し、手持ちの情報で進める
   - 設定済みの場合: 以下のコマンドでツイート検索を実行
   ```bash
   curl -s -H "Authorization: Bearer ${X_API_BEARER_TOKEN}" \
     "https://api.x.com/2/tweets/search/recent?query={テーマ}&max_results=20&tweet.fields=created_at,public_metrics,text" \
     | python3 -c "
   import json, sys
   data = json.load(sys.stdin)
   for tweet in data.get('data', []):
       print(f\"[{tweet.get('created_at','')}] {tweet['text']}\")
       metrics = tweet.get('public_metrics', {})
       print(f\"  likes: {metrics.get('like_count',0)}, RT: {metrics.get('retweet_count',0)}\")
       print()
   "
   ```
   - エンゲージメント（いいね・RT数）が高い投稿を優先的にピックアップする

6. 全Phaseの結果を統合し、以下のフォーマットでまとめる:

```
## Xリサーチ結果

情報ソース: {WebSearch / Nitter / X API}（使用したPhaseを記載）

### トレンド・話題
- {箇条書きで3〜5件}

### 注目の意見・切り口
- {箇条書きで3〜5件}

### 使えそうなフレーズ・表現
- {箇条書きで3〜5件}

### 高エンゲージメント投稿（X API使用時のみ）
- {いいね・RT数が多い投稿の要約}
```

7. 情報ソースを明記する（どのPhaseで取得したか）
8. リサーチ結果を表示した上で、AskUserQuestionツールで確認:
  question: "Xリサーチ結果を確認してください。このまま進めますか？"
  header: "リサーチ確認"
  options:
    - label: "このまま進める（推奨）"
      description: "この結果をベースにステップ③へ進みます"
    - label: "追加で調べてほしい"
      description: "キーワードや切り口を追加してリサーチし直します"
    - label: "リサーチ結果を使わない"
      description: "Xリサーチを省略してRAG検索のみで進めます"

**承認を得てから次へ進む。**

## ステップ⑤ X投稿文を6パターン生成

**やること:**
1. ステップ②〜④で集めた情報をすべて統合する
2. 以下のプロンプトに従い、6つの投稿文を生成する:

[prompt]
あなたはSNSマーケターでありインフルエンサーとして、メッセージ性の強い独り言投稿文章を6つ様々な方法で下記ルールに則って作成しなさい。文章は詰めて書きなさい。

#ルール
・投稿の6構成パターンをそれぞれ守った上で、トピックに沿った悩み解決文章を6つ違うパターンで作る。
・[禁止]文章はハッシュタグや絵文字を禁止
・【文字数制限（最重要・厳守）】Xの投稿上限は「合計280ポイント」。全角文字（漢字・ひらがな・カタカナ・全角記号）=1文字あたり2ポイント、半角文字（英数字・半角記号）=1文字あたり1ポイント、絵文字=2ポイント。全角のみなら最大140文字、半角のみなら最大280文字、混在時はポイント合計が280以内であればOK。投稿文は合計280ポイント以内に必ず収めること。改行や絵文字を使用せず書きなさい。
・ネガティブな入りでポジティブに終わらせる
・なるべく6つの投稿はランダムに、それぞれに違いを出してください。

※親しみやすく、タメ口で、自然な話し方で"ひとりごと"のように話しなさい

#構成
1.[強い興味訴求フックワード][結論に関連する具体的な物語][端的な方法][結論]

2.[強烈な結論][具体的手段1,2,3,,,][まとめ結論]

3.[結論][ベネフィットの提示][理由][独り言]

4.[機能的内容][情緒的内容][強烈なフックワード]

5.[昔の自分の悩みの打ち明け][ストーリー][今の変化した自分]

6.[トピックの否定文章][理由をたて肯定意見に変更][ベネフィットの打ち出し][キャッチーで強烈なワードで体言止め]

文章は詰めて書きなさい。合計280ポイント以内（厳守）で書きなさい。

詰め字で投稿用の文章として書き出して。

例：経営者はこれを常に頭に入れておけ。数字は嘘をつかない。でも、すべてを語ってはくれない。数字は「結果」でしかなく、「原因」はいつも人と現場の中にある。上手くいっている経営者は必ず現場にも足を運び、リアルな声を聞いているんだよ。経営者は、現場の声も必ず拾うようにすると上手くいくぞ
[/prompt]

3. 各投稿文の文字数をBashツールで正確にカウントする（python3で全角・半角を区別してカウント）:
   ```bash
   python3 -c "
   import unicodedata
   text = '''投稿文をここに入れる'''
   count = 0
   for c in text:
       if unicodedata.east_asian_width(c) in ('F', 'W'):
           count += 2
       else:
           count += 1
   print(f'{len(text)}文字, {count}/280pt {\"OK\" if count <= 280 else \"OVER!\"}')
   "
   ```
   280ポイントを超えている場合は、必ず短縮してから出力すること。
4. 各投稿文に文字数を表記する
4. 以下のフォーマットで出力:

```
## X投稿文 6パターン

テーマ: 「{テーマ}」

---

### パターン1 [興味訴求フック型] ({N}文字)
{投稿文}

### パターン2 [強烈な結論型] ({N}文字)
{投稿文}

### パターン3 [結論→ベネフィット型] ({N}文字)
{投稿文}

### パターン4 [機能×情緒型] ({N}文字)
{投稿文}

### パターン5 [ビフォーアフター型] ({N}文字)
{投稿文}

### パターン6 [否定→肯定型] ({N}文字)
{投稿文}
```

## 注意事項

- 各ステップで必ず **AskUserQuestionツール** でユーザーの承認を得てから次に進むこと（テキストで「よろしいですか？」と聞くのではなく、選択肢UIを使う）
- Xリサーチ=OFFの場合、ステップ②③をスキップしてステップ④に進む
- 【最重要】投稿文は合計280ポイント以内を厳守すること（全角=2pt、半角=1pt、絵文字=2pt）。Xの投稿上限。必ずBashでpython3を使ってポイント数を検証してから出力する
- ハッシュタグ・絵文字・改行は絶対に使わない
- タメ口で自然な独り言調を維持する
- 山本さんのRAGデータから具体的な数字やエピソードを積極的に活用する
